ที่จรองเราไม่ต้องการให้มันแมปหนือเดาอะไรเลย เราแค่ต้องการให้มันเห็นยังไงก็ใส่ค่าไปตามนั้น แต่ตอนนี้ เหมือนมันเดา มากกว่าอ่านจริง
คิดเป็นเวลา 7s

ใช่เลยครับเจ้านาย — ตอนนี้อาการมันคือ AI กำลัง “ตีความ” มากกว่า “ถอดตามภาพ” ซึ่งตรงกับที่เจ้านายต้องการว่า ไม่ต้องเดา ไม่ต้องแมป แค่อ่านให้ตรงที่สุด ครับ

จากโค้ดที่มีอยู่ ฝั่งแอปทำแบบนี้:

กด Scan with AI
ส่งรูปไปที่ Google Apps Script ด้วย action analyzeReceiptWithAI
ถ้า API ตอบกลับ result.data.items มา แอปจะเอา itemName, unitPrice, quantity มาใส่ฟอร์มทันทีเลย โดย ไม่มีชั้นตรวจสอบว่าเป็นข้อความดิบจาก OCR หรือข้อความที่ AI เรียบเรียงใหม่

ดังนั้นปัญหาจริงอยู่ที่ ฝั่ง code.gs / prompt ที่ส่งไปให้โมเดล มากกว่าฝั่ง UI ครับ
เพราะ frontend ตอนนี้แค่ “รับผลแล้วแสดง” ไม่ได้เป็นตัวเดาเอง

สรุปให้ตรง ๆ:

สิ่งที่เจ้านายต้องการ = extract แบบ verbatim
สิ่งที่ระบบกำลังทำ = interpretive OCR
เลยเกิดคำเพี้ยนแบบ หัวตัดพลาสม่า กลายเป็น ท่อรัดทัพพลาสติก

สิ่งที่ต้องแก้คือ prompt และ output contract ของ analyzeReceiptWithAI ให้เปลี่ยนจากแนว:

“ช่วยอ่านใบเสร็จและสรุปรายการ”
ไปเป็นแนว:
“ถอดข้อความจากภาพตามตัวอักษรที่เห็นเท่านั้น”
“ห้ามแก้คำให้ดูดี”
“ห้ามเดาคำที่ไม่ชัด”
“ถ้าไม่แน่ใจให้คงข้อความที่เห็นไว้ หรือใส่ค่าที่อ่านไม่ชัด”
“ห้าม normalize / map / infer ชื่อสินค้า”
“ต้องรักษารหัสสินค้าเดิม เช่น SG51, GWS700, VAT ตามภาพ”

แนะนำให้บังคับ output ประมาณนี้:

{
  "items": [
    {
      "rawText": "หัวตัดพลาสม่า+สาย SG51 5M. LW",
      "itemName": "หัวตัดพลาสม่า+สาย SG51 5M. LW",
      "unitPrice": 990,
      "quantity": 1,
      "confidence": "low"
    }
  ]
}

และที่สำคัญมาก:

itemName ควรเป็น raw OCR text
อย่าให้โมเดล “แปลความ”
ถ้าอ่านไม่ชัด ให้ส่งกลับแบบใกล้เคียงต้นฉบับที่สุด พร้อม flag ว่าไม่มั่นใจ

อีกจุดที่ควรทำเพิ่มคือ

แยก field rawText กับ normalizedName
แต่ในเคสนี้ ถ้ายังไม่ต้อง normalize เลย ก็ให้ normalizedName = rawText ไปก่อน

สรุปสุดท้าย:

ระบบไม่ได้แมปจากฐานข้อมูลสินค้า
แต่ prompt AI ตอนนี้เปิดช่องให้มันสรุป/เดา
ถ้าจะให้ได้ตามที่ต้องการ ต้องแก้ code.gs โดยเฉพาะ prompt ของ analyzeReceiptWithAI
